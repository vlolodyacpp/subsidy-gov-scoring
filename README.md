# Subsidy Gov Scoring

Система автоматического скоринга заявок на государственные субсидии в сельском хозяйстве Казахстана.

## Архитектура

```
FastAPI backend  ←→  Streamlit dashboard
     ↓
Правиловая оценка (17 факторов, 6 групп)
     +
ML-модель (опционально, подключается автоматически)
     +
Advisory (исторические данные)
```

## Быстрый старт

```bash
# Установка зависимостей
pip install -r requirements.txt
```

Запуск — **два терминала одновременно**:

```bash
# Терминал 1 — FastAPI backend (порт 8000)
python main.py --serve

# Терминал 2 — Streamlit дашборд (порт 8501)
streamlit run app/start.py
```

После запуска:
- Дашборд: http://localhost:8501
- API документация (Swagger): http://localhost:8000/docs

## Скоринговая модель

### Правиловый скоринг (17 факторов, 6 групп)

| Группа | Факторы | Суммарный вес |
|--------|---------|---------------|
| **Нормативное соответствие** | normative_match, amount_normative_integrity, amount_adequacy | 20% |
| **Бюджет и очередь** | budget_pressure, queue_position | 22% |
| **Региональная специфика** | region_specialization, region_direction_approval_rate, akimat_approval_rate | 20% |
| **Характеристики заявки** | unit_count, direction_approval_rate, subsidy_type_approval_rate | 10% |
| **Условия содержания** | pasture_compliance, mortality_compliance, grazing_utilization | 20% |
| **Регуляторная сложность** | criteria_complexity, direction_risk, regional_pasture_capacity | 8% |

Все факторы имеют значения [0, 1]. Итоговый балл: сумма `значение × вес × 100` = [0, 100].

### ML-модель (опционально)

Если в директории `models/` присутствует обученная модель (`bundle.pkl`), движок автоматически переключается в режим `merit-ml-advisory`:

- Правиловый скор рассчитывается как обычно
- ML-модель предсказывает вероятность одобрения на основе всех признаков
- Итоговый балл = blend(rule_score, ml_score) с настраиваемыми весами
- SHAP-объяснения отображаются на вкладке "Оценка на основе исторических данных"

### Advisory (историческая рекомендация)

На основе похожих заявок из истории (регион + направление) система формирует текстовую рекомендацию: "история поддерживает" / "история предупреждает".

## API

### Справочники

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/regions` | Список регионов |
| GET | `/directions` | Направления субсидирования |
| GET | `/subsidy-types?direction=X` | Типы субсидий (с фильтром) |
| GET | `/districts?region=X` | Районы (с фильтром) |
| GET | `/akimats?region=X` | Акиматы (с фильтром) |

### Скоринг

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/score` | Оценка одной новой заявки |
| POST | `/rank` | Ранжирование заявок из датасета |
| GET | `/explain/{app_id}` | Детальный анализ заявки по номеру |

### Статистика

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/stats` | Агрегированная статистика |
| GET | `/factor-stats` | Статистика по факторам |
| GET | `/region-factors` | Факторы по регионам |
| GET | `/timeline` | Динамика по времени |

### Системные

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/health` | Статус API |
| POST | `/upload-dataset` | Загрузка нового датасета XLSX |
| GET | `/retrain-status` | Статус последнего переобучения |

### Пример запроса к `/score`

    "amount": 780000,
    "submit_month": 3,
    "submit_day": 15

## Дашборд — страницы

### 🏛️ Обзор
Общая картина: метрики (всего заявок, средний балл, одобрено), гистограмма распределения баллов, топ регионов по одобряемости.

### 📋 Шортлист
Ранжированная таблица заявок с переключателем 50/100/200. Экспорт в CSV. Показывает правиловый и ML-балл если модель загружена.

### 🔎 Детали
Глубокий анализ конкретной заявки: факторный водопад, группировка по 6 блокам, ML-объяснения (SHAP), заключение для комиссии.

### ⚖️ Сравнение
Сравнение двух заявок по всем факторам в виде горизонтального барчарта.

### 📊 Аналитика
Статистика по факторам, тепловая карта регионов, временная динамика.

### 📝 Новая заявка
Форма для оценки входящей заявки в режиме реального времени. Каскадные дропдауны: Регион → Район/Акимат, Направление → Тип субсидии. После нажатия "Оценить" показывает полный факторный анализ как на странице Деталей.

## Данные

Основной датасет: `data/subsidies.xlsx` — заявки на субсидии из государственного портала. Поля: номер заявки, регион, район, акимат, направление, тип субсидии, сумма, норматив, количество единиц, дата подачи, статус.

Нормативы Приложения 1 (максимальные суммы субсидий по типам) хранятся в `data/cleaned/subsidy_normatives.csv`. Парсинг из XLS — `src/normatives.py`.

Синтетические признаки для ML (`data/cleaned/synthetic_features.csv`): pasture_compliance, mortality_compliance, grazing_utilization, criteria_complexity, direction_risk, regional_pasture_capacity — сгенерированы с seed=42, условно коррелируют с is_approved.
