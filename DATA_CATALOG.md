## 2. Существующие вычисляемые признаки (features.py)

**Формат:** все признаки нормализованы в [0, 1], без leakage (expanding window).

### 2.1. Соответствие нормативам (26% весов в rule-based)

| Признак | Описание |
|---------|----------|
| `normative_match` | Совпадение суммы с эталонным нормативом (точное или ±5%) |
| `amount_normative_integrity` | Делится ли сумма нацело на норматив (целое кол-во единиц) |
| `amount_adequacy` | Адекватность суммы относительно медианы по типу субсидии (Z-score) |

### 2.2. Бюджет и очередь (28%)

| Признак | Описание |
|---------|----------|
| `budget_pressure` | Доля одобренных сумм от запрошенных по (регион, направление) на момент подачи |
| `queue_position` | Позиция в очереди: `1 / (1 + log(prior_count))` |

### 2.3. Региональный контекст (31%)

| Признак | Описание |
|---------|----------|
| `region_specialization` | Доля направления в общем потоке заявок региона |
| `region_direction_approval_rate` | Одобряемость (регион, направление) |
| `akimat_approval_rate` | Общая одобряемость акимата |

### 2.4. Характеристики заявки (15%)

| Признак | Описание |
|---------|----------|
| `unit_count` | Перцентильный ранг количества единиц внутри типа субсидии |
| `direction_approval_rate` | Одобряемость направления |
| `subsidy_type_approval_rate` | Одобряемость типа субсидии |

### 2.5. Дополнительные признаки (используются только ML, не в rule-based)

| Признак | Описание |
|---------|----------|
| `normative_original_match` | Совпадение суммы с оригинальным нормативом из заявки |
| `normative_reference_gap` | Отклонение от эталонного норматива (clipped 0-3) |
| `normative_reference_typicality` | Лог-расстояние от эталона (centered 0.5) |
| `amount_to_type_median_ratio` | Сумма / медиана по типу (clipped 0-5) |
| `unit_count_original_log` | log(количество единиц по оригинальному нормативу) |
| `region_approval_rate` | Региональная одобряемость (shrunk 35% к глобальной) |
| `direction_history_count_log` | log(кол-во прошлых заявок по направлению) |
| `submit_month_sin` / `submit_month_cos` | Циклическое кодирование месяца |

## 5. Как всё соединить для обучения

```python
import pandas as pd
from src.pipeline import load_raw_data, clean_data
from src.features import build_feature_tables, extract_features_batch

# 1. Основной датасет + существующие 11 признаков
df = clean_data(load_raw_data("data/subsidies.xlsx"))
tables = build_feature_tables(df)
features = extract_features_batch(df, tables)

# 2. Присоединить синтетические признаки
synth = pd.read_csv("data/cleaned/synthetic_features.csv")
features = features.merge(synth, on="app_number", how="left")

# 3. Target
y = features["is_approved"]
X = features[FEATURE_COLUMNS]  # выбрать нужные колонки
```

## 6. Известные ограничения данных

1. **Дисбаланс классов 10.9:1** — необходим class_weight="balanced" или oversampling
2. **Козоводство — 3 заявки** — слишком мало для обучения, может засорить модель
3. **Синтетические данные** — pasture/mortality/grazing сгенерированы, не реальные. При замене на реальные данные корреляции могут измениться
4. **Нормативы не покрывают все типы** — для неизвестных типов `normative` = 0
5. **Temporal structure** — заявки упорядочены по времени, при разбиении на train/test нужен temporal split, не random
6. **г. Шымкент** — замаплен на Туркестанскую область для пастбищных норм (нет отдельных данных)
