# Subsidy Scoring System

**Кейс 2 — Скоринг сельхозпроизводителей при получении субсидий**
---
## Проблема

Сегодня субсидии на развитие животноводства в Казахстане распределяются по принципу **«первый подал — первый получил»**: финансирование уходит тому, кто раньше собрал документы, пока не закончился бюджет. Реальная эффективность хозяйства, история субсидирования и потенциал к росту — не учитываются.

Министерство сельского хозяйства хочет перейти к **merit-based скорингу** — распределению субсидий на основе данных о производителях.

## Решение

AI-система, которая **ранжирует заявки** по набору факторов, **объясняет каждый score** (explainability) и **формирует shortlist** для рассмотрения комиссией. Финальное решение остаётся за человеком — система рекомендует, не решает.

### Что делает система

1. Загружает реестр заявок (36 651 запись) из xlsx
2. Извлекает 9 признаков из каждой заявки (feature engineering)
3. Рассчитывает score 0–100 через ML-модель (CatBoost) + весовую формулу
4. Генерирует текстовое объяснение: какие факторы повлияли и насколько (SHAP)
5. Формирует отсортированный shortlist с рекомендациями
6. Отображает результаты в дашборде (Streamlit)

## Признаки (features)

Из сырых данных заявки извлекаются 9 числовых признаков:

| # | Признак | Описание | Обоснование |
|---|---------|----------|-------------|
| 1 | `region_approval_rate` | % одобренных заявок в регионе | Региональная специфика бюджетов и комиссий |
| 2 | `direction_approval_rate` | % одобрений по направлению | Приоритетные направления субсидирования |
| 3 | `subsidy_type_approval_rate` | % одобрений по типу субсидии | Востребованность конкретных программ |
| 4 | `amount_adequacy` | Близость суммы к медиане для данного типа | Завышенные суммы чаще отклоняются |
| 5 | `season_approval_rate` | % одобрений в месяц подачи | Бюджет исчерпывается к концу года |
| 6 | `district_activity` | Нормализованная активность района | Районы с высокой активностью — развитое с/х |
| 7 | `amount` | Запрошенная сумма субсидии | Абсолютный размер запроса |
| 8 | `normative` | Норматив субсидирования (тенге) | Установлен Правилами для каждого вида |
| 9 | `amount_to_norm_ratio` | Соотношение суммы к нормативу | Кол-во единиц в заявке |

## Модель скоринга

Система использует **гибридный подход** — комбинацию экспертных весов и ML-модели:

### v1: Weighted rule-based (этап 1)

Экспертные веса, основанные на Правилах субсидирования:

| Фактор | Вес | Почему |
|--------|-----|--------|
| Региональная одобряемость | 25% | Бюджеты и комиссии различаются по регионам |
| Одобряемость направления | 20% | Государственные приоритеты в животноводстве |
| Одобряемость типа субсидии | 20% | Критерии различаются по видам субсидий |
| Адекватность суммы | 20% | Аномальные суммы — признак проблемной заявки |
| Сезонность подачи | 15% | Ранние заявки чаще одобряются (бюджет не исчерпан) |

**Валидация v1:** средний score одобренных заявок = 86.5, отклонённых = 84.0.

### v2: CatBoost + SHAP (этап 2)

CatBoost обучается на бинарном таргете `is_approved` (одобрена/исполнена = 1, отклонена/отозвана = 0). Преимущества: нативная работа с категориями, устойчивость к переобучению, интеграция с SHAP.

## Explainability

Каждый score сопровождается объяснением. Пример вывода:

```
Score: 95.4/100 (Риск: Низкий)

✓ Региональная одобряемость: высокий (97%) — +24.1 баллов
✓ Адекватность суммы заявки: высокий (100%) — +20.0 баллов
✓ Одобряемость типа субсидии: высокий (94%) — +18.9 баллов
✓ Одобряемость направления: высокий (92%) — +18.4 баллов
● Сезонность подачи: средний (93%) — +14.0 баллов
```

На этапе 2 добавятся SHAP waterfall-графики — визуализация вклада каждого фактора в конкретный score.

## Сценарии использования

**Аналитик МСХ формирует shortlist:** загружает заявки → система рассчитывает scores → отсортированный shortlist → передаёт комиссии.

**Член комиссии проверяет заявку:** выбирает заявку в дашборде → видит score + SHAP-график → понимает обоснование → принимает решение.

**Региональное управление анализирует:** видит распределение scores по региону → выявляет слабые направления → корректирует работу.
